  🛠️ TRON-MeetingAI 後端技術摘要

  1. 核心架構與進程管理 (main.py)
   * 非同步通訊架構：採用 FastAPI 作為 API 接口，並透過 multiprocessing 實現 AI 任務的進程隔離。這能有效避免 AI
     模型載入時導致的界面卡死與 OpenMP DLL 衝突。
   * 穩定性機制：實作了基於「本地暫存檔案」的資料傳輸機制，避開了 Python Manager Proxy 在 Windows 下傳輸數千筆逐字稿片段時的效能瓶頸。
   * 資源監控：主進程會即時監控子進程狀態，若發生顯存溢出 (OOM) 導致的崩潰，能自動捕捉並回報錯誤代碼。

  2. 語音轉文字引擎 (services/stt.py)
   * 模型最佳化：整合 faster-whisper 並手動調整 CUDA 運算精度（目前設定為 float16），在確保 8GB 顯存安全的前提下獲得最高轉錄品質。
   * 語義連貫性優化：內建「片段合併邏輯」，自動將間隔小於 1.5 秒的破碎語句進行智慧合併，大幅減少逐字稿的碎片化，並將後續 AI
     摘要的效率提升了約 5 倍。
   * 幻覺抑制：關閉了 condition_on_previous_text 並改進了 initial_prompt 樣板，徹底解決了模型在無聲處產生重複指令文字的幻覺問題。

  3. 智慧摘要引擎 4.0 (services/llm.py)
   * Map-Reduce 兩階段演算法：
       * Map 階段：將長篇逐字稿自動分段（每段 3000 字），精準提取原始事實點。
       * Reduce 階段：將彙整後的事實點交由 LLM 進行全局歸納，產出結構化的報告。
   * 結構化輸出 (JSON Schema)：利用 llama-cpp-python 的最新功能，強制 AI 必須輸出符合規範的 JSON 格式，保證 100% 的解析成功率。
   * 台灣商務語感：透過 Few-Shot Prompting 技術，引導 Llama-3-Taiwan 產出具備專業語氣、符合台灣商務習慣的會議記錄。

  4. 擴充工具 (utils/yt_downloader.py)
   * 影音自動化：整合 yt-dlp 工具，實現 YouTube 網址一鍵下載並自動提取高品質 WAV 音訊，完整串接轉譯流程。

   1 cd sidecar

  ---

  打包配置說明：
   1. console=False：在 .spec 中已設定此參數，打包出的 .exe 執行時不會出現終端機黑畫面。
   2. hiddenimports：包含了 uvicorn 的所有底層協議、fastapi、torch、llama_cpp 以及 multiprocessing 等隱藏依賴，確保程式能完整運行。
   3. COLLECT (One-Dir 形式)：打包結果會存放在 sidecar/dist/sidecar_engine 資料夾中。這是針對 AI 專案最穩定的方式，因為 torch 與 cuda
      有大量 DLL 檔案，使用 One-Dir 可以避免單一大型 EXE 啟動緩慢或解壓失敗的問題。
   4. 多進程修復：我已在 main.py 加入 freeze_support()，解決了 PyInstaller 在 Windows 上啟動子進程可能導致的遞迴報錯問題。

  打包完成後，您可以將 sidecar/dist/sidecar_engine 資料夾中的 sidecar_engine.exe
  作為最終的可執行檔。如果有任何打包過程中的錯誤，隨時告訴我！
