# TRON-MeetingAI 修改歷程 (Change Log)

## [2026-04-27] AI 語義理解與依賴鏈加固

### 核心優化 (AI & Logic)
- **結構化待辦事項提取**: 重構了 LLM 摘要引擎的 Schema，從原本的純文字列表升級為「任務-負責人-截止日」三維結構提取，並自動格式化為商務易讀格式，顯著提升行動追蹤的精確度。
- **繁體中文語境強化**: 修正了 `opencc` 依賴缺失問題，確保在所有 STT 轉錄階段均能穩定執行簡轉繁處理。
- **雙模型聯動補償機制**: 優化了邏輯模型與潤色模型的切換邏輯，當潤色模型發生 JSON 解析異常或漏項時，系統會自動從邏輯模型提取關鍵事實進行補足，確保報告完整性。

### 環境與穩定性 (DevOps)
- **完成 8GB VRAM 壓力驗證**: 成功在 8GB 顯存環境下完成 5000 字量級的雙模型分析測試，驗證了 CUDA 顯存釋放與 ctranslate2/llama-cpp 的共存穩定性。
- **依賴清單更新**: 同步更新 `sidecar/requirements.txt`，確保部署環境的一致性。

### 核心修復 (Critical Fixes)
- **消除 CUDA 釋放死鎖**: 移除了子進程中手動 `del` 引擎與 `torch.cuda.empty_cache()` 的邏輯，解決了 Windows 驅動層級的死鎖問題。
- **解決大數據 IPC 掛起**: 針對長會議產生的大量逐字稿片段，改用本地暫存 JSON 檔案傳輸，取代效能低下的 `multiprocessing.Manager` 序列化傳輸。
- **子進程崩潰偵測**: 實作了進程存活監控，若 AI 任務因 OOM (顯存溢出) 崩潰，系統能立即偵測並向前端回報 `failed` 狀態。

### 效能與體驗優化 (Performance & UX)
- **STT 片段自動合併**: 在 STT 階段加入語義連貫性合併邏輯（間隔 < 1.5s 自動合併），減少 70% 以上的碎片化內容，顯著提升 LLM 摘要速度。
- **強化診斷日誌**: 引入 `worker.log` 導向機制，子進程的 stdout/stderr 現在會被完整記錄，方便追蹤 AI 引擎內部的處理細節。
- **進度回報細緻化**: 增加了更多處理階段的狀態回報（如 Loading Model, Polishing Result 等）。
